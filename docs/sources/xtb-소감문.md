## 앱 제작의 배경 및 동기

### 1. 실험 설계 과정에서의 물질 선정 문제 인식

학교 동아리 프로젝트와 과제연구를 진행하면서, 실험 목적에 적합한 후보 물질을 선정하는 과정이 단순한 문헌 조사나 직관적 판단만으로는 충분하지 않다는 문제를 인식하였다.

새로운 가설을 실험으로 검증하기 위해서는 후보 물질의 작용 메커니즘, 안전성, 제작 가능성, 실험 조건 등을 함께 고려해야 한다. 그러나 후보군이 늘어날수록 각 물질을 동일한 기준으로 비교하기 어렵고, 실험 전에 우선순위를 정하는 과정에서 판단의 불확실성이 커진다.

특히 표면 화학, 흡착, 여과와 같이 분자 수준의 정전기적 상호작용이 결과에 영향을 줄 수 있는 실험에서는 이러한 문제가 더욱 두드러진다. 겉보기에는 적합해 보이는 물질이라도 실제 분자 구조, 전하 분포, 극성, 안정성에 따라 전혀 다른 실험 결과가 나타날 수 있다. 따라서 실험 후보 물질을 선정할 때 정성적 추측뿐 아니라 **계산 기반의 정량적 근거**를 함께 활용할 필요가 있다고 판단하였다.

### 2. 생성형 AI를 활용한 물질 선정 메커니즘의 구조적 한계

최근에는 ChatGPT와 같은 거대언어모델(LLM)을 활용하여 연구 아이디어나 후보 물질을 탐색하는 사례가 늘어나고 있다. LLM은 다양한 문헌적 지식과 일반적 패턴을 바탕으로 빠르게 후보군을 제안할 수 있다는 장점이 있으나, 본질적으로 학습 데이터의 통계적 패턴을 기반으로 문장을 생성하는 모델이기 때문에 분자의 기하 최적화나 전자구조 계산 작업에는 다소 오류가 발생한다.

이로 인해 존재하지 않는 물질명, 근거가 불충분한 물성 수치, 실제 계산 없이 제시된 안정성 판단 등이 추천 결과에 포함될 위험이 있다. 이러한 할루시네이션을 그대로 실험 설계에 반영하면 시간과 비용이 낭비될 수 있으며, 안전성 측면에서도 문제가 발생할 수 있다.

따라서 본 프로젝트에서는 LLM을 **계산 화학 도구가 산출한 정량 결과를 해석하고 검토하는 보조 계층**으로 활용하는 방향으로 설정하였다.

### 3. 문제 해결 방안: MAS (Multi-Agent System)

본 프로젝트의 핵심 목표는 계산 화학 엔진과 LLM 기반 멀티에이전트 시스템을 결합하여, 실험 후보 물질을 보다 근거 있게 선별하는 시스템을 설계하는 것이다. 이를 위해 다음과 같은 구조를 적용하였다.

1. **xTB**를 활용하여 후보 물질별 총 에너지, 부분 전하, 쌍극자 모멘트, 구조 최적화 수렴 여부 등 정량적 물리화학 지표를 산출한다.
2. 역할이 분리된 **전문 에이전트**가 동일한 xTB 계산 결과를 바탕으로 각자의 관점에서 후보 물질을 평가한다.
3. **Critic Agent**가 과잉 해석, 근거 부족, 방법론적 한계를 검토한다.
4. 최종적으로 **가중치 기반 점수**와 **실험 설계 보고서**를 자동 생성한다.

즉, 본 프로젝트는 LLM을 “실험 후보를 임의로 추천하는 모델”이 아니라, 외부 계산 도구에 의해 산출된 결과를 해석하고 검토하는 **구조화된 판단 계층**으로 배치하였다. 이를 통해 실험 후보 물질 추천 과정에서 발생할 수 있는 할루시네이션을 줄이고, 계산 근거에 기반한 의사결정 보조 시스템을 구현하고자 하였다.

## 수행 결과 · 결과물 설명

### [화학 시뮬레이터 제작]

### 1. xTB 계산 화학 엔진 래핑 및 데이터 파이프라인 구조 설계

본 프로젝트에서는 xTB의 **GFN2-xTB** 계산 방식과 구조 최적화 옵션을 기본 조건으로 사용하였다. 후보 물질은 사전에 준비한 `.xyz` 형식의 3차원 분자 구조 파일로 입력하였고, xTB 계산 결과를 바탕으로 각 후보의 총 에너지, 부분전하, 쌍극자 모멘트, 구조 최적화 수렴 여부를 추출하였다.

구현 과정에서는 xTB 명령어를 Python에서 실행하는 **실행 모듈**과 계산 결과 로그에서 필요한 수치를 추출하는 **해석 모듈**을 분리하였다.

- 실행 모듈: xTB 계산을 자동으로 호출
- 해석 모듈: 출력된 계산 로그에서 필요한 값을 읽어 구조화된 데이터로 변환
- 저장 형식: 정해진 형식의 **JSON** 데이터

| 항목 | 의미 |
|------|------|
| 총 에너지 | 후보 물질의 상대적 안정성 비교에 활용 |
| 최대 양전하 / 음전하 | 정전기적 상호작용 가능성 판단에 활용 |
| 쌍극자 모멘트 | 분자의 극성 정도 비교에 활용 |
| 구조 최적화 수렴 여부 | 계산 결과와 신뢰성 판단에 활용 |
| 원시 로그 경로 | 계산 근거 추적 및 재검토에 활용 |

초기 프로토타입에서는 xTB가 출력하는 긴 텍스트 로그를 그대로 에이전트에 전달하였다. 그러나 이 방식에서는 수치를 잘못 읽거나, 단위를 혼동하거나, 일부 항목을 누락하는 문제가 발생하였다. 이를 해결하기 위해 계산 로그를 직접 사용하는 대신, 해석 모듈이 필요한 수치만 추출하여 엄격한 JSON 형식으로 변환하도록 수정하였다. 이를 통해 에이전트가 동일한 계산 결과를 같은 형식으로 읽을 수 있게 되었고, xTB 계산 결과를 정리한 구조화 데이터만 바탕으로 판단하도록 데이터 처리 흐름을 고정하여 계산 근거의 추적 가능성도 확보할 수 있었다.

### 2. MCP 기반 도구 서버 설계

에이전트가 xTB 계산을 직접 호출할 수 있도록 본 프로젝트에서는 **MCP(Model Context Protocol)** 기반 도구 서버를 설계하였다. MCP는 외부 도구를 거대언어모델이나 에이전트 실행 환경과 연결하기 위한 통신 방식이다. 본 프로젝트에서는 이를 활용하여 에이전트가 xTB 계산 기능을 하나의 도구처럼 사용할 수 있도록 하였다.

| 도구 이름 | 역할 |
|-----------|------|
| `calculate_candidate` | 단일 `.xyz` 파일에 대해 xTB 계산을 수행하고 결과를 JSON으로 반환 |
| `calculate_candidate_batch` | 여러 후보 물질을 한 번에 계산 |
| `generate_evidence_table` | 후보별 주요 계산 결과를 표로 정리 |
| `evaluate_candidates` | xTB 계산 결과를 바탕으로 에이전트 평가 수행 |
| `run_experiment_pipeline` | 입력 파일부터 계산·평가·보고서 생성까지 전체 흐름 실행 |
| `generate_experiment_report` | 평가 결과를 바탕으로 실험 설계 보고서 작성 |

위 구조의 핵심은 **계산 → 해석 → 평가 → 보고서 작성** 단계를 분리한 것이다. MCP 서버는 xTB 계산 실행, 결과 해석, 구조화 데이터 생성, 규칙 기반 평가를 담당한다. 반면 Gemini API와 같은 LLM 기반 서술 보강 기능은 선택적으로 사용할 수 있도록 분리하였다.

이를 통해 시스템의 핵심 판단이 LLM의 문장 생성에 의존하지 않도록 하였다. 수치 계산과 후보 순위는 xTB 계산 결과와 규칙 기반 평가에서 결정되고, LLM은 필요한 경우 보고서 문장을 자연스럽게 다듬는 보조 역할만 수행하도록 설계하였다.

### 3. 멀티에이전트 구조 및 교차 검토 방식

단일 LLM이 모든 판단을 수행하면 특정 관점에 치우치거나 근거가 부족한 내용을 확정적으로 제시할 위험이 있다. 이를 줄이기 위해 본 프로젝트에서는 역할이 분리된 멀티에이전트 구조를 적용하였다. 각 에이전트는 동일한 xTB 계산 결과를 입력으로 받지만, 서로 다른 관점에서 후보 물질을 검토한다.

| 에이전트 | 담당 역할 |
|----------|-----------|
| 총괄 에이전트 | 전체 계산 및 평가 흐름 조율 |
| 가설 정리 에이전트 | 실험 목적을 검증 가능한 가설로 정리 |
| 후보 물질 에이전트 | 후보군의 역할과 실험 적합성 검토 |
| 정전기 상호작용 에이전트 | 부분전하와 쌍극자 모멘트를 바탕으로 정전기적 상호작용 가능성 평가 |
| 비공유 상호작용 에이전트 | 분산력 등 약한 상호작용 경향 검토 |
| 구조 안정성 에이전트 | 구조 최적화 수렴 여부와 상대적 안정성 평가 |
| 안전성 에이전트 | 실험 전 1차 안전성 검토 |
| 변인 통제 에이전트 | 독립변인·종속변인·통제변인 정리 |
| 비판 검토 에이전트 | 근거 부족·과잉 해석·방법론적 한계 검토 |

기본 구현은 API 키 없이 작동하는 **Python 규칙 기반 평가** 방식으로 구성하였다. 각 에이전트는 xTB 계산 결과를 바탕으로 점수, 요약, 추천 상태를 반환한다. 추천 상태는 `include`, `caution`, `exclude`로 구분하였다.

특히 **구조 안정성 · 안전성 · 비판 검토** 중 하나라도 `exclude`를 반환하면 해당 후보는 최종 추천에서 제외되도록 하였다. 이는 단순히 점수가 높은 후보를 선택하는 방식이 아니라, 실험 적용 전 반드시 걸러야 할 위험 요소를 우선적으로 반영하기 위한 장치이다.

선택적으로 Gemini API를 사용할 경우 보고서 후반부의 서술을 보강할 수 있도록 하였다. 그러나 수치·순위·제외 판단의 기준은 항상 xTB 계산 결과와 규칙 기반 평가에 두었다.

### 4. 가중치 기반 채점 방식 및 보고서 자동화

에이전트별 평가 결과를 최종 후보 선정에 활용하기 위해 가중치 기반 채점식을 적용하였다. 본 프로젝트의 실험 목적은 풍화 미세플라스틱 표면과 필터 활성층 사이의 정전기적 상호작용 가능성을 비교하는 것이므로, **정전기적 상호작용 항목에 가장 높은 가중치**를 부여하였다.

보고서에는 다음이 포함되도록 하였다.

- 실험 목적
- xTB 계산 범위와 한계
- 후보 목록
- xTB 계산 결과 표
- 에이전트별 평가 결과
- 제외 후보 사유
- 최종 순위
- 상위 후보 추천
- 변인 통제
- 예상 실패 요인

또한 동일한 수치가 여러 곳에 반복되어 서로 다르게 기록되는 문제를 막기 위해, 계산 결과는 **하나의 JSON 데이터를 기준으로만** 불러오도록 하였다.

### 5. 사례 적용: 풍화 미세플라스틱 제거 필터 후보 선별

본 시스템을 동아리에서 진행 중인 풍화 미세플라스틱 제거 실험에 적용하였다.

**핵심 가설:** 풍화된 미세플라스틱 표면이 음전하 또는 극성을 띨 수 있으며, 이와 정전기적으로 상호작용할 수 있는 양전하 필터 활성층을 사용하면 제거 가능성이 높아질 수 있다.

실제 미세플라스틱 입자 전체를 계산 대상으로 삼지 않고, 미세플라스틱 표면을 단순화한 **대리 구조**와 필터 활성층을 단순화한 대리 구조를 비교하는 방식으로 범위를 제한하였다.

**필터 활성층 후보:** 키토산 코팅, PEI 코팅, 4차 암모늄 유도체, 양이온 전분, 폴리도파민 유사 코팅 등  
**대조군:** 중성 셀룰로오스, 방향족 활성탄 대리 구조

입력 파일에는 후보 조합명, 3차원 구조 파일 경로, 형식, 전하, 후보 역할, 대조군 여부를 기록하였다. 이후 xTB 계산을 통해 각 후보의 부분전하, 쌍극자 모멘트, 구조 최적화 수렴 여부를 확인하였다.

시스템의 전체 점수표에서는 대조군이 높은 점수를 받을 수 있었지만, 최종 추천 후보 선정에서는 실험 목적에 직접 대응하는 **양전하 필터 활성층 후보만**을 대상으로 하였다.

| 순위 | 후보 |
|------|------|
| 1순위 | 키토산 코팅 조합 |
| 차순위 | 4차 암모늄 유도체, 폴리도파민 유사 코팅 |
| 대조군 | 추천이 아닌 비교 기준으로 유지 |

보고서는 각 후보의 계산 결과, 대조군 처리 이유, 추천 후보 선정 근거, 방법론적 한계, 후속 실험 설계, 예상 실패 요인을 포함한 실험 설계 초안을 생성하였다. 특히 **xTB 결과가 실제 여과 효율을 직접 의미하지 않으며, 분자 수준 대리 구조를 비교하기 위한 초기 선별 지표**라는 점을 명시하였다.

## 사회적 기여도 · 기대 효과

본 프로젝트는 과학 탐구에서 생성형 AI를 사용할 때 발생할 수 있는 할루시네이션 문제를 줄이는 하나의 방법을 보여준다. LLM이 임의로 후보 물질을 추천하는 방식이 아니라, xTB 계산 결과를 먼저 산출하고 그 결과를 기준으로 에이전트가 검토하도록 설계했기 때문이다.

이 구조의 장점은 **계산 근거와 서술 생성이 분리**된다는 점이다. 후보 물질의 수치 결과와 순위는 xTB 계산 및 규칙 기반 평가에서 결정되고, LLM은 필요한 경우 이를 사람이 읽기 쉬운 보고서 형태로 정리하는 역할을 맡는다.

또한 본 시스템은 학교 실험 환경에서 후보 물질을 사전에 좁히는 데 활용할 수 있다. 제한된 시간과 예산 안에서 모든 후보를 직접 실험하기는 어렵기 때문에, 계산 기반 초기 선별을 통해 우선 실험할 후보를 정하는 것은 실험 설계의 효율성을 높일 수 있다.

다만 본 시스템은 실험 결과를 완전히 예측하는 도구가 아니다. xTB 계산은 분자 수준 대리 구조를 대상으로 한 초기 선별 도구이며, 실제 여과 효율은 입자 크기, 표면 거칠기, 용매 조건, pH, 농도, 교반 조건, 필터 제작 방식 등에 의해 달라질 수 있다. 따라서 본 시스템은 최종 결론을 대신하는 도구가 아니라, **실험 전 후보 물질의 우선순위를 정하고 판단 근거를 구조화하는 의사결정 보조 시스템**으로 활용해야 한다.

## 개인 성찰

### 인상 깊었던 부분

새롭게 학습한 계산 화학과 멀티에이전트 기술이 실제 학교 실험의 문제를 해결하는 도구로 발전하는 과정이 가장 흥미로웠다. 특히 동아리에서 진행한 미세플라스틱 제거 필터 실험에 직접 적용하고 후보별 판단 근거와 우선순위가 생성되는 과정을 확인할 때 가장 재미있었다. 서로 다른 분야인 화학과 AI를 하나의 시스템 안에서 연결할 수 있다는 점도 인상적이었다.

### 고심·시행착오

가장 오래 고심한 부분은 복잡한 xTB 계산 결과를 에이전트가 오류 없이 이해할 수 있는 데이터로 변환하는 것이었다. 초기에는 계산 로그에서 수치를 잘못 추출하거나 단위를 혼동하고 일부 항목을 누락하는 문제가 발생하였다. 이를 해결하기 위해 필요한 값만 추출하는 규칙을 정하고, 모든 결과를 엄격한 JSON 형식으로 변환하도록 파싱 과정을 수정하였다.

### 배운 점 · 성장

이 프로젝트를 통해 화학과 AI를 유기적으로 연결하는 과정을 경험하였다. 특히 자연어 기반 거대언어모델이 갖는 정성적 추론의 한계를 데이터로 제어하는 과정에서 다양한 데이터를 융합하는 인터페이스 설계 능력을 향상시킬 수 있었다. 계산 화학 엔진의 산출물을 정형화하고 다중 에이전트 간의 엄격한 교차 검증망을 구축함으로써, 복잡한 시스템 내에서 발생하는 데이터 오염이나 오류를 논리적으로 추적하고 통제하는 소프트웨어 공학적 문제 해결 역량을 체득하였다.

### 후속 탐구 질문

- 멀티에이전트를 활용한 화학 시뮬레이터를 제작하고 싶다. 각 분자 간의 상호작용에 대해 깊이 공부하고 각 AI에 적용하여 이를 하나의 시스템으로 이어주면, 첨가하는 물질에 따라 반응을 예측해주는 AI를 제작할 수 있지 않을까.
- 현재는 외부 시뮬레이터에 의존하여 제작하였지만, 추후 분자와 원자·전자에 이해가 깊어진다면 가상 실험실을 만들어 미래 연구에 도움이 될 것이라 기대된다.
- 논문 탐색 중 알게 된 **뉴로심볼릭 AI**에 대해 자세히 알고 탐구하고 싶다. (블랙박스 문제 — 사고와 논리의 근거를 인간이 명확하게 알 수 없음 — 를 해결한다고 평가되는 차세대 AI)
