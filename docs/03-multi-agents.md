# 03 · 멀티에이전트

## 공통 입력

모든 전문 에이전트는 **동일한 xTB 결과 JSON**을 받는다. 관점만 다르다.

## 에이전트 역할

| 에이전트 | 역할 |
|----------|------|
| 총괄 | 전체 계산·평가 흐름 조율 |
| 가설 정리 | 실험 목적을 검증 가능한 가설로 정리 |
| 후보 물질 | 후보 역할·실험 적합성 |
| 정전기 상호작용 | 부분전하·쌍극자 기반 정전기 상호작용 가능성 |
| 비공유 상호작용 | 분산력 등 약한 상호작용 경향 |
| 구조 안정성 | 수렴 여부·상대 안정성 |
| 안전성 | 실험 전 1차 안전성 |
| 변인 통제 | 독립·종속·통제변인 정리 |
| 비판 검토 (Critic) | 근거 부족·과잉 해석·방법론적 한계 |

## 출력 계약 (에이전트당)

각 에이전트는 최소 다음을 반환한다.

- `score` — 수치 점수
- `summary` — 짧은 근거 요약
- `status` — `include` | `caution` | `exclude`

## 하드 제외 규칙

다음 중 **하나라도** `exclude`이면 해당 후보는 최종 추천에서 **제외**한다.

- 구조 안정성
- 안전성
- 비판 검토 (Critic)

점수가 높아도 제외를 뒤집을 수 없다.

## 가중치

실험 목적이 “풍화 미세플라스틱 ↔ 필터 활성층 **정전기 상호작용**”인 사례에서는 **정전기 상호작용 항목 가중치가 최대**이다.

다른 실험으로 일반화할 때는 목적에 맞게 가중치만 재정의하고, 제외 규칙은 유지하는 것을 기본으로 한다.

## LLM과의 관계

- 기본: **Python 규칙 기반**, API 키 불필요
- 선택: Gemini 등으로 보고서 **서술만** 보강
- 수치·순위·제외는 규칙 결과를 따른다
