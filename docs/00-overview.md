# 00 · 프로젝트 개요

## 한 줄 요약

**xTB 계산(정량) → 구조화 JSON → 규칙 기반 멀티에이전트 평가 → 실험 설계 보고서**  
(+ 선택: LLM으로 문장 다듬기) + **홍보용 Next.js 웹페이지**

## 프로젝트명

xTB를 활용한 계산 화학 결과와 멀티에이전트 검증을 활용한 실험 후보 물질 추천 시스템 개발 및 홍보 웹 페이지 제작

## 해결하려는 문제

1. 실험 후보 물질이 많을 때 **동일 기준으로 비교·순위화**하기 어렵다.
2. LLM만으로 후보를 추천하면 **할루시네이션**(존재하지 않는 물질, 근거 없는 수치) 위험이 있다.
3. 표면 화학·흡착·여과처럼 **전하·극성·안정성**이 중요한 실험에서는 정량 근거가 필요하다.

## 설계 원칙 (에이전트 필수 준수)

1. **LLM은 추천의 근원이 아니다.** LLM은 계산·규칙 결과를 해석·서술하는 보조 계층이다.
2. **수치의 단일 출처는 xTB → 파서 → JSON**이다. 보고서/UI는 이 JSON만 참조한다. 로그 원문을 LLM에 그대로 넣지 않는다.
3. **제외 우선:** 구조 안정성 / 안전성 / Critic 중 하나라도 `exclude`면 최종 추천에서 제외한다.
4. **xTB는 실측 예측기가 아니다.** 대리 구조(proxy) 기반 초기 선별 지표임을 문서·UI·보고서에 명시한다.
5. **계산·해석·평가·보고서** 단계를 분리한다. 한 모듈이 전부 하지 않는다.

## 기술 스택 (목표)

| 영역 | 기술 |
|------|------|
| 계산 | xTB (기본: GFN2-xTB + 구조 최적화) |
| 입력 | `.xyz` 3D 구조 |
| 래핑 | Python 실행 모듈 + 로그 해석 모듈 |
| 도구 인터페이스 | MCP 도구 서버 |
| 평가 | API 키 없이도 동작하는 Python 규칙 기반 멀티에이전트 |
| LLM (선택) | Gemini API 등 — 보고서 문장 보강만 |
| 웹 홍보 | Next.js (본 저장소) |

## 비목표 (Out of scope로 취급)

- 전체 미세플라스틱 입자 규모 시뮬레이션
- LLM만으로 후보 순위를 매기는 흐름
